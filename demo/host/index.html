Maintainer's Copilot — Demo Host Page

This page simulates an external website that has embedded the Maintainer's Copilot chat widget. The chat bubble appears in the bottom-right corner once the backend is running and a valid widget ID is set.

Before opening this page:
  1. Start the backend: docker compose up -d
  2. Build the widget: cd widget && npm install && npm run build
  3. Register a user via the Streamlit app or:
    curl -X POST http://localhost:8000/api/v1/auth/register -H "Content-Type: application/json" -d '{"email":"admin@example.com","password":"yourpassword"}'
  4. Create a widget config via the Streamlit admin page or the admin API, copy the public_widget_id UUID.
  5. Replace 99a70ab8-cbff-4615-9b46-f0f0da8064b7 in the <script> tag at the bottom of this file.
  6. Open this file directly in a browser — no build step required for the host page.

Simulated page content

This is a typical host page. The Maintainer's Copilot chat widget floats in the bottom-right corner. Click the bubble to expand the chat panel.

The widget fetches its configuration (greeting, theme, enabled tools) from GET /api/v1/widgets/{public_widget_id} at load time and calls POST /api/v1/chat for every user message.

How embedding works

Paste one <script> tag anywhere in your page's <body>. The loader script at /widget.js:

  1. Reads data-widget-id and data-api-base-url from the tag.
  2. Creates an <iframe> pointing to the React widget app (/widget-app/).
  3. Listens for postMessage events from the widget to resize the iframe when the user opens or closes the chat panel.
  4. Keeps widget CSS and JavaScript isolated from the host page via the iframe boundary.

Embed snippet

<script
  src="http://localhost:8000/widget.js"
  data-widget-id="99a70ab8-cbff-4615-9b46-f0f0da8064b7"
  data-api-base-url="http://localhost:8000"
></script>
