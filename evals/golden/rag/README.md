# RAG Golden Set

## Status

**RAG-3 COMPLETE** — `rag_golden.jsonl` finalized (25 examples, docs=5, issue=10, comment=10).

## Purpose

A 25-example hand-curated golden set for evaluating the RAG pipeline
(retrieval quality + answer quality) on kubernetes/kubernetes issues.

Separate from the classifier golden set (`evals/golden/classification_golden.jsonl`).

## Candidate CSV Schema

| Column | Description |
|---|---|
| `candidate_id` | Stable ID (`cand_0000` format) |
| `question` | Template-generated question for curator review |
| `ideal_answer` | Extractive answer snippet from the source chunk |
| `ground_truth_chunk_ids` | Semicolon-separated chunk_id values from section-aware chunks |
| `source_urls` | GitHub URL(s) for the source material |
| `source_types` | `docs`, `issue`, or `comment` |
| `issue_numbers` | Issue number if applicable |
| `notes` | Generator notes: source, labels, section, char_length |
| `hand_labeled_for_judge_check` | Default `false`; set `true` for at least 5 in final set |
| `curator_status` | Default `candidate`; curator sets to `selected` for final 25 |

---

## Final Schema (`rag_golden.jsonl`)

Each record must have:

| Field | Type | Description |
|---|---|---|
| `question` | string | The maintainer question to answer |
| `ideal_answer` | string | Expected answer text (hand-written or corrected from candidate) |
| `ground_truth_chunk_ids` | list[str] | IDs of chunks that contain the answer |
| `source_urls` | list[str] | GitHub or docs URLs for the ground-truth source |
| `notes` | string | Curator notes: why this question, ambiguities |
| `hand_labeled_for_judge_check` | bool | True if used for LLM judge agreement audit |

---

## Requirements

- **25 total examples** across question types: bug triage, feature requests, docs clarification, how-to.
- **At least 5 examples** must have `hand_labeled_for_judge_check: true` for LLM judge agreement audits.
- All `ground_truth_chunk_ids` must exist in `data/rag/chunks/chunks_section_aware.jsonl`.
- No issue_number from the RAG golden set may overlap with the classifier splits (`data/processed/`).

## Review Workflow (start here)

1. Open `rag_golden_candidates_review.csv` — this is the file to edit.
2. Read the `answer_preview` column — single-line, noise-stripped preview of the answer.
3. Check `suggested_keep` (yes / maybe / no) and `suggested_reason` for each row.
4. For rows marked **yes**: accept or refine question and ideal_answer; set `selected_for_final = yes`.
5. For rows marked **maybe**: evaluate manually; fix noise in ideal_answer if worth keeping; set `selected_for_final = yes` if you decide to keep it.
6. For rows marked **no**: skip unless you have a specific reason to override.
7. Target: exactly **25 rows** with `selected_for_final = yes`.
8. Set `hand_labeled_for_judge_check = true` for at least **5 rows** (used to audit LLM judge agreement).
9. Copy corrected `question` and `ideal_answer` values back into the original `rag_golden_candidates.csv` if you want to keep the original in sync.
10. Run `pipelines/rag/finalize_golden.py` to validate and write `rag_golden.jsonl`.

## Review Column Descriptions

| Column | Description |
|---|---|
| `answer_preview` | Max-300-char single-line preview of ideal_answer (badges, shortcodes, code fences stripped) |
| `suggested_keep` | `yes` = clean and useful; `maybe` = useful but needs manual cleanup; `no` = log-heavy, vague, or code-only |
| `suggested_reason` | Brief reason for the suggestion |
| `selected_for_final` | Set to `yes` for your 25 chosen rows |
| `reviewer_notes` | Free-form notes for yourself during review |

## Constraint

> **ground_truth_chunk_ids must point to real chunks in `data/rag/chunks/chunks_section_aware.jsonl`.**
> Do not create `rag_golden.jsonl` before curation is complete.
